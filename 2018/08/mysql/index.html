Mysql

August 10, 2018 in linux server, mysql

本篇文章讲解mysql在linux下的编译安装以及主从部署

Table of Contents

  • 下载最新稳定版mysql
  • 下载，解压，转移到安装目录
  • 开始安装

下载最新稳定版mysql

  • 点击进入mysql的下载地址
  • 选择最下边的社区版mysql MySQL Community Edition
  • 并在里面选择MySQL Community Server版本下载

根据自己的linux系统选择相应的版本
先查看linux 系统

    root@iZuf6ixwbfspqnitk9f37oZ:/data# cat /etc/issue
    Ubuntu 16.04.3 LTS \n \l

再查看linux系统是多少位

    root@iZuf6ixwbfspqnitk9f37oZ:/data# getconf LONG_BIT
    32

下载，解压，转移到安装目录

根据服务器系统和位数选择下载的包，不过网上推荐下载通用安装方法的TAR包
wget http://cdn.mysql.com/Downloads/MySQL-5.7/mysql-5.7.21-linux-glibc2.12-i686.tar.gz

解压，转移到安装目录

开始安装

  1. 首先检查mysql用户的创建

    查看mysql用户是否已经创建

        cat /etc/group | grep mysql
    cat /etc/passwd | grep mysql
    ###如果没有mysql用户，进行创建
    groupadd mysql
    useradd -r -g mysql mysql
    #useradd -r参数表示mysql用户是系统用户，不可用于登录系统

  2. 数据库数据存放路径 以及文件夹权限

        mkdir /data/mysql
    chown -R mysql:mysql /data/mysql
    chown -R mysql:mysql /alidata/server/mysql

  3. 初始化数据库

    我下载的是5.7.21版本数据库，需要使用bin/mysqld 初始化
    老版本使用scripts/mysql_install_db 现已弃用

    bin/mysqld --user=mysql --basedir=/alidata/server/mysql --datadir=/data/mysql  --initialize
2018-02-03T09:05:09.930363Z 0 [Warning] TIMESTAMP with implicit DEFAULT value is deprecated. Please use --explicit_defaults_for_timestamp server option (see documentation for more details).
2018-02-03T09:05:12.180375Z 0 [Warning] InnoDB: New log files created, LSN=45790
2018-02-03T09:05:12.390451Z 0 [Warning] InnoDB: Creating foreign key constraint system tables.
2018-02-03T09:05:12.527946Z 0 [Warning] No existing UUID has been found, so we assume that this is the first time that this server has been started. Generating a new UUID: 578ba22f-08c1-11e8-946a-00163e1c55ee.
2018-02-03T09:05:12.531811Z 0 [Warning] Gtid table is not ready to be used. Table 'mysql.gtid_executed' cannot be opened.
2018-02-03T09:05:12.542928Z 1 [Note] A temporary password is generated for root@localhost: D;g/V%XXXXX
#记录给出的root密码

  4. 配置mysql

    将mysql配置成服务，设置开机启动

        cp support-files/mysql.server /etc/init.d/mysql
    chmod +x /etc/init.d/mysql
    update-rc.d -f mysql defaults 
        
    #vim /etc/init.d/mysql 修改内容
    basedir=/alidata/server/mysql
    datadir=/data/mysql
    上传my.cnf到/etc做全局设置
        
    vim /etc/profile
    #新增/alidata/server/mysql/bin
    PATH=$PATH:/alidata/server/mysql/bin
    export PATH
        
    #编辑后重新加载
    source /etc/profile

  5. 登录mysql

        # 第一次登录
    mysql -u root -p
    #输入上面记录的root@localhost密码
    #成功登录
        
    #如果忘记了root密码，只能使用/mysqld_safe登录去修改密码
    server mysql stop
    mysqld_safe --user=mysql --skip-grant-tables --skip-networking & 
    mysql -u root mysql
    mysql> UPDATE user SET Password=PASSWORD('newpassword') where USER='root';
    mysql> FLUSH PRIVILEGES;
    mysql> quit;
    #重启mysql
    server mysql start
    mysql -u root -p
    #输入newpassword
    #成功登录
        
    #首次登入需要修改密码
    SET PASSWORD = PASSWORD('123456'); 
        
    # 创建新用户
    create user 'root'@'%' identified by 'K8$g@S19WTN*7j4p';  
    #赋权，所有权限
    GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY 'K8$g@S19WTN*7j4p' WITH GRANT OPTION;
    #重新载入赋权表  
    FLUSH PRIVILEGES;
    #查看用户权限
    show grants for root;
